HTML+CSS实现彩色文件夹切换效果网页

初学HTML和CSS，了解了一些基本概念之后觉得还是通过搞事学习比较有意思

然后发现了一个神奇好用的前端案例网站 http://thecodeplayer.com/

基于里面的一个例子进行理解和模仿，最终生成了一个具有文件夹页面切换效果的网页 (部分版本的IE浏览器可能无法正常显示)

以下是简要的制作流程：

HTML

这两天稍微理解了一些前端语言之间的关系，如果把制作网页看成是搭一间房子，那么HTML作为描述网页结构的工具，就是钢筋混凝土骨架，而CSS样式表则是进行精细的内部装修。因此，首先要完成HTML的构建。这个例子中，网页实际可操作的内容并不多，实际上主要元素只是左侧的4个选择区和右侧的显示块主体。选择区可以用radio单选框实现，而右侧的主体部分实际上可以视为互相覆盖的4个块，即4个不同的div。因此，HTML的内容并不复杂（然而即使这么简单后来还是发现了bug，接下来会解释）：

所以接下来CSS的内容才是关键。首先既然要将div作为占满屏幕的块，那么就要将其样式设置为占满全屏。通过nth-of-type选择器为4个div块分别设置不同底色，由于目前并没有切换效果，所以第4个div块会覆盖其他元素并占满全屏。

这里有几个比较关键的属性。


  • outline和outline-offset

outline属性指定了边框的样式，而outline-offset属性设置边框的偏移量。当outline-offset为-10时，相当于边框在原边框基础上向内收缩了10px，而outline又被设置为宽度为10px的solid填充，因此这两个属性实际上起到了纯色填充的效果。至于具体填充什么颜色，交由nth-of-type选择器进行设置。

为了更好地理解这两个属性的作用，尝试把outline-offset设置为-5，效果如下：





可以很明显地看出，相对于20px的宽度，-5px的缩进只能填充一半的内容。


  • z-index

z-index我个人觉得可以用类似于ppt中“向下一层”、“置于顶层”的观点进行理解。z-index大的元素优先置于顶层，这里将单选框的z-index置为1或者任意正数皆可，因不存在其他z-index更高的元素。因为单选框的z-index大于div,因此左侧的4个选择区始终会出现在div块上方而不会被覆盖。如果将z-index设置成0，那么屏幕上又只有单调的纯色块了。


  • cursor

当鼠标放到选择区时，应把箭头图标变为小手图标以进行指示，将cursor设置为pointer即可起到这一效果。

CSS：切换效果

在CSS中，切换使用transform属性,这里我们只需要使用transform中的translateX（沿X轴方向平移）即可。如transform: translateX(-100%)表示将元素向左平移一个窗口的距离，也就是完全移出去。这里的括号内的内容应该始终是相对于初始窗口的位置而言的，即对于一个translateX(-100%)的元素，执行translate(0%)并不表示停在向左移动后的位置不动，而是向右平移，回到其向左移动前在窗口中的原始位置。

因为transform的效果十分唐突，因此一般都配合transition的过渡效果配合使用。使用

transition:transform 0.4s ease 0.4s

可以添加时长为0.4秒，时间曲线为ease(规定慢速开始，然后变快，然后慢速结束的过渡效果)的过渡效果。
